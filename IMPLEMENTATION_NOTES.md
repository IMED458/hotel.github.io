# ცვლილებების ჩანაწერი

## არსებული ფუნქციები

- შენარჩუნებულია არსებული static PMS სტრუქტურა: `index.html`, `assets/js/app.js`, `assets/css/styles.css`.
- შენარჩუნებულია კალენდრის არსებული ვიზუალური/ინტერაქტიული ლოგიკა: დღე/კვირა/თვე, drag/drop, ფილტრები, სრული ეკრანი და ჯავშნის დეტალები.
- შენარჩუნებულია არსებული ძირითადი მონაცემები `localStorage`/Data SDK სტრუქტურით: rooms, reservations, guests, payments.

## დამატებული მოდულები

- როლებზე დაფუძნებული ნავიგაცია და გაფართოებული როლები: მფლობელი, გენერალური მენეჯერი, ოპერაციები, რეცეფცია, ჯავშნების ოპერატორი, ღამის აუდიტორი, შემოსავლების მენეჯერი, housekeeping, maintenance, ბუღალტერი, HR და სისტემის ადმინისტრატორი.
- ახალი გვერდები: დალაგება, ტექნიკური მომსახურება, ერთიანი დავალებები, დამტკიცებები, დოკუმენტები, თანამშრომლები და აუდიტის ჟურნალი.
- ერთიანი audit log (`auditLog`) და ფინანსურ audit log-თან დაკავშირება.
- Housekeeping task-ები და maintenance ticket-ები საწყისი დემო მონაცემებით.
- Approval workflow-ის საწყისი სტრუქტურა.
- Document და employee ჩანაწერების საწყისი სტრუქტურები.
- Firebase Firestore სინქი: ყველა `setState` ცვლილება ინახება `hotelState` კოლექციაში, ხოლო არსებული localStorage მონაცემები პირველ ჩატვირთვაზე Firestore-ში მიგრირდება.

## უსაფრთხო გაუმჯობესებები

- ჯავშნის წაშლა შეიცვალა გაუქმებაზე: ჩანაწერი რჩება ისტორიაში.
- გადახდის წაშლა შეიცვალა reversal/correction ჩანაწერზე: ორიგინალი გადახდა არ იშლება.
- ხარჯის წაშლა შეიცვალა reversal/correction ჩანაწერზე.
- Check-out-ის შემდეგ ნომერი გადადის დასალაგებელ სტატუსზე.
- დასალაგებელ/ტექნიკურ სტატუსში მყოფ ნომერზე check-in-ისას ჩნდება გაფრთხილება.

## ქართული UI

- ნავიგაცია, როლები, სტატუსები, room type label-ები და ფინანსური tab-ების ნაწილი გადაყვანილია ქართულად.
- დარჩენილი ტექნიკური შიდა კოდები, key-ები, endpoint-ები და integration ტერმინები შენარჩუნებულია საჭიროებისამებრ.

## შემდეგი რეკომენდებული ეტაპი

1. `assets/js/app.js` დაიყოს მოდულებად: `state.js`, `rbac.js`, `calendar.js`, `finance.js`, `operations.js`, `modals.js`.
2. დაემატოს რეალური backend და migration სისტემა, რადგან მიმდინარე პროექტი static/localStorage არქიტექტურაა.
3. დაინერგოს სრულფასოვანი i18n key-value ფაილი ქართულ default ენაზე.
4. დაემატოს automated UI regression ტესტები კალენდრისთვის.
